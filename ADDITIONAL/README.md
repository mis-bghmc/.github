# ADDITIONAL Issue Templates

This directory contains additional GitHub issue templates that can be used for various types of tasks and reports. These templates help standardize issue creation and ensure all necessary information is captured.

## Templates

| File | Name | Description |
|------|------|-------------|
| `01-task.yml` | **Task** | Create a new task with a task items checklist. Use this for general tasks that require multiple steps to complete. |
| `02-task-item.yml` | **Task Item** | Create a sub-task item under a main task. Use this to break down larger tasks into smaller, manageable pieces. |
| `03-bug-report.yml` | **Bug Report** | Report a bug to help improve the system. Includes steps to reproduce, expected vs actual behavior, and logs/screenshots. |
| `04-enhancement.yml` | **Enhancement** | Detail an improvement to an existing feature that will be implemented. Includes affected area, implementation plan, and acceptance criteria. |
| `05-modification.yml` | **Modification** | Document changes to existing features or functions. Requires a reason for the modification and a checklist of items. |
| `06-weekly-review.yml` | **Weekly Review** | Report weekly findings including bugs, enhancements, tasks, and other observations. Useful for regular team reviews. |
| `07-documentation.yml` | **Documentation** | Use this template for documentation tasks and content updates. Helps track what part of the documentation is being updated. |
| `08-issue-compilation.yml` | **Issue Compilation** | Create issues with multiple contents or simply a compilation of related items. Useful for grouping related tasks together. |

## Usage

When creating a new issue in GitHub, you can select any of these templates from the dropdown menu. Fill in the required fields (marked with `*`) and any optional fields as needed.

## Note

These templates are **still subject to changes, enhancements, improvements, and discussions**. They may be updated over time to better serve the project's needs. Feel free to suggest modifications or provide feedback on how these templates can be improved.
